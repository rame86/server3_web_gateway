# 프로젝트 프론트엔드 구조 요약

현재 프론트엔드는 `server/web/client` 디렉토리에 위치해 있으며, **React (Vite 기반)** 환경에서 작성되었습니다. 주요 라우팅은 `wouter`를 사용하고 있으며 스타일링은 Tailwind CSS와 Radix UI(Shadcn UI 계열) 컴포넌트를 사용하고 있습니다.

## 주요 디렉토리 구조

```text
client/
├── package.json       # 프로젝트 의존성 관리 (React, Vite, Tailwind CSS, Radix UI 등)
├── vite.config.js     # Vite 빌드 설정
└── src/               # 실제 프론트엔드 소스 코드가 위치하는 폴더
    ├── App.jsx        # 전체 애플리케이션의 라우팅(wouter) 설정 (경로에 따른 페이지 연결)
    ├── main.jsx       # React 앱의 진입점 (DOM 랜더링)
    ├── index.css      # 글로벌 스타일 및 Tailwind 지시어
    ├── components/    # 재사용 가능한 UI 컴포넌트 모음
    │   ├── ui/        # 버튼, 모달, 입력창 등 기본 UI 요소 (Radix UI 기반)
    │   ├── Layout.jsx # 페이지의 기본 레이아웃 (헤더, 사이드바 등)
    │   └── TopNav.jsx # 상단 네비게이션 바
    ├── pages/         # 경로(URL)별로 보여지는 개별 페이지 컴포넌트들
    │   ├── admin/     # 관리자용 페이지 (대시보드, 유저 관리 등)
    │   ├── artist/    # 아티스트용 페이지 (정산, 팬덤 관리 등)
    │   ├── auth/      # 인증 관련 페이지 (로그인, 회원가입)
    │   ├── user/      # 일반 유저용 페이지 (스토어, 예매, 커뮤니티, 지갑 등)
    │   ├── Home.jsx   # 메인(랜딩) 페이지
    │   └── NotFound.jsx # 404 에러 페이지
    ├── contexts/      # 상태 관리를 위한 React Context Api (예: Theme 관리)
    ├── hooks/         # 커스텀 React 훅 모음
    └── lib/           # 각종 유틸리티 함수나 설정 (axios 인스턴스 등)
```

## 컴포넌트와 라우팅 구조 이해 (`App.jsx` 기준)

프론트엔드의 구조는 크게 역할(Role)에 따라 페이지가 나뉘어 있습니다.
`App.jsx` 파일에서 이 라우팅 구조를 확인할 수 있습니다.

1. **공통 영역 (Landing & Auth)**
   - `/` : 홈 페이지 (`Home.jsx`)
   - `/login` : 로그인 (`UserLogin.jsx`)
   - `/signup` : 회원가입 (`UserSignup.jsx`)

2. **일반 유저 (User)**
   - `/user` : 유저 대시보드
   - `/user/store`, `/user/booking`, `/user/chat`, `/user/community`, `/user/wallet` 등 유저 기능별 페이지

3. **아티스트 (Artist)**
   - `/artist` : 아티스트 대시보드
   - `/artist/fandom`, `/artist/settlement` 등 관리 기능

4. **관리자 (Admin)**
   - `/admin` : 관리자 대시보드
   - `/admin/users`, `/admin/artists`, `/admin/store`, `/admin/settlement` 등 전체 관리 기능

이러한 구조를 통해 각 권한별로 필요한 화면(`pages` 하위)이 별도로 분리되어 있고, 공통으로 사용하는 디자인 요소들(버튼, 레이아웃 등)은 `components`에서 가져다 쓰는 효율적인 구조로 구성되어 있습니다.
