# 프로젝트 프론트엔드 구조 요약

현재 프론트엔드는 `server3_web_gateway` 내부에 구성되어 있습니다. 메인 앱은 `client` 디렉토리에 위치해 있으며, **React (Vite 기반)** 환경에서 작성되었습니다. 주요 라우팅은 `wouter`를 사용하고 있으며 스타일링은 Tailwind CSS와 Radix UI 및 사용자 지정 컴포넌트를 혼합해 사용하고 있습니다.

## 주요 디렉토리 구조

```text
server3_web_gateway/
├── client/              # 실제 동작하는 React 프론트엔드 메인 폴더
│   ├── package.json     # 프로젝트 의존성 관리 (React, Vite, Tailwind CSS 등)
│   ├── vite.config.js   # Vite 빌드 설정
│   └── src/             # 어플리케이션 소스 코드
│       ├── App.jsx      # 전체 애플리케이션의 라우팅(wouter) 설정 (경로에 따른 페이지 연결)
│       ├── main.jsx     # React 앱의 진입점 (DOM 랜더링)
│       ├── index.css    # 글로벌 스타일 및 Tailwind 지시어
│       ├── components/  # 재사용 가능한 UI 컴포넌트 모음 (Layout 등)
│       ├── pages/       # 경로(URL)별로 보여지는 개별 페이지 컴포넌트들
│       │   ├── admin/   # 관리자용 페이지 (대시보드, 회원/아티스트 관리, 상점 관리 등)
│       │   ├── artist/  # 아티스트용 페이지 (정산, 팬덤 관리, 상점 관리 등)
│       │   ├── auth/    # 인증 관련 페이지 (로그인, 회원가입)
│       │   ├── user/    # 일반 유저용 페이지 (스토어, 예매, 커뮤니티, 지갑 등)
│       │   │   ├── UserStore.jsx           # 스토어 메인 목록
│       │   │   ├── UserStoreDetail.jsx     # 스토어 상품 상세
│       │   │   └── UserPurchaseProcess.jsx # 상품 구매/결제 진행 프로세스
│       │   ├── Home.jsx                    # 메인(랜딩) 페이지
│       │   └── NotFound.jsx                # 404 에러 페이지
│       ├── contexts/    # 상태 관리를 위한 React Context Api (예: Theme 관리)
│       ├── hooks/       # 커스텀 React 훅 모음
│       └── lib/         # 각종 유틸리티 함수나 API 설정 (axios 인스턴스, shopApi 등)
├── shared/              # 프론트엔드와 다른 서비스 간 공유될 수 있는 리소스나 설정 파일 포맷
├── package.json         # Workspace 전체 구성용
└── components.json      # Shadcn-ui 또는 컴포넌트 관리 용 설정
```

## 컴포넌트와 라우팅 구조 이해 (`App.jsx` 기준)

프론트엔드의 구조는 크게 역할(Role)에 따라 계층을 나누어 페이지를 제공합니다.
`App.jsx` 파일에서 이 라우팅 구조를 확인할 수 있습니다.

1. **공통 영역 (Landing & Auth)**
   - `/` : 홈 페이지 (`Home.jsx`)
   - `/login` : 로그인 (`UserLogin.jsx`)
   - `/signup` : 회원가입 (`UserSignup.jsx`)

2. **일반 유저 (User)**
   - `/user` : 유저 대시보드
   - `/user/store` : 유저 대상 상점 스토어 메인 목록 (`UserStore`)
   - `/user/store/:id` : 유저 상품 상세 화면 (`UserStoreDetail`)
   - `/user/store/purchase/:id` : 상품 구매 및 포인트 결제 (`UserPurchaseProcess`)
   - `/user/booking`, `/user/chat`, `/user/community`, `/user/wallet` 등 기능별 페이지 제공

3. **아티스트 (Artist)**
   - `/artist` : 아티스트 대시보드
   - `/artist/store` : 아티스트 상점 관리 기능
   - `/artist/fandom`, `/artist/settlement` 등 권한별 관리 기능

4. **관리자 (Admin)**
   - `/admin` : 관리자 대시보드
   - `/admin/store` : 관리자 상점 및 굿즈 관리
   - `/admin/users`, `/admin/artists`, `/admin/settlement` 등 전체 관리 기능

이러한 구조를 통해 각 권한별로 필요한 화면(`pages` 하위)이 별도로 분리되어 있고, API 통신은 `src/lib/api.js` 내부에서 개별 서비스별 Axios 인스턴스(`shopApi`, `payApi` 등)를 통해 마이크로서비스 백엔드와 연결됩니다.
